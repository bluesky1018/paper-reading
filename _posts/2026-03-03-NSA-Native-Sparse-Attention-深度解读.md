## 一句话总结

> NSA 是一种**原生可训练的稀疏注意力机制**，通过动态分层稀疏策略（压缩 + 选择 + 滑窗）与硬件对齐的内核优化，在 64k 长序列上实现了高达 **11.6× 解码加速**和 **9.0× 前向加速**，同时在通用基准、长上下文和推理任务上**超越了全注意力基线**。

## 背景与动机

长上下文建模是下一代大语言模型的核心能力。从 OpenAI 的 o-系列模型到 DeepSeek-R1，再到 Gemini 1.5 Pro，业界正在将上下文长度从几千推到几十万 token。然而，标准 softmax 注意力的 $O(n^2)$ 复杂度带来了巨大的计算瓶颈：

- **解码阶段**：处理 64k 上下文时，注意力计算占总延迟的 **70-80%**
- **训练阶段**：长序列训练的计算成本急剧增长

现有的稀疏注意力方法存在两大痛点：

1. **"高效推理的幻觉"**：很多方法理论上减少了计算量，但实际加速效果有限。部分方法只加速解码不加速预填充（如 H2O），或只加速预填充不加速解码（如 MInference）。更关键的是，很多方法与 GQA（分组查询注意力）等现代架构不兼容——虽然减少了计算量，但由于 KV-cache 是组内共享的，内存访问量并没有减少。

2. **"可训练稀疏的迷思"**：大多数稀疏注意力方法只在推理阶段使用，而预训练时仍用全注意力。这意味着模型并未学习到如何高效利用稀疏模式。尝试将现有方法用于训练时，要么涉及不可微操作（如 K-means 聚类），要么因为 token 级选择导致非连续内存访问，无法适配 FlashAttention 等高效实现。

## 架构/方法详解

### 整体框架

NSA 的核心思想是将原始的全长度 KV 序列替换为一组**更紧凑、信息更密集**的代表性 KV 对。具体来说，对于每个查询 $\mathbf{q}_t$，NSA 通过三条并行注意力路径处理前文的 K、V：

![NSA 整体架构图。左图展示三条并行注意力路径：压缩注意力捕获粗粒度全局模式，选择注意力关注重要 token 块，滑窗注意力处理局部上下文。右图展示三种注意力模式的可视化，绿色区域为需要计算注意力分数的区域，白色区域可跳过。](/assets/img/nsa/x2.png)
*图 1：NSA 架构总览。三条并行路径分别处理不同粒度的注意力计算，绿色区域表示需计算的注意力，白色区域可跳过。*

最终输出通过门控机制加权融合：

$$\mathbf{o}_t^* = \sum_{c \in \mathcal{C}} g_t^c \cdot \text{Attn}(\mathbf{q}_t, \tilde{K}_t^c, \tilde{V}_t^c)$$

其中 $\mathcal{C} = \{\text{cmp}, \text{slc}, \text{win}\}$ 分别代表压缩、选择和滑窗三种策略，$g_t^c$ 是通过 MLP + Sigmoid 生成的门控分数。

### 1. Token 压缩（Compression）

将连续的 token 块通过可学习的 MLP 压缩为单个代表性 token：

- **块大小 $l=32$**，滑动步长 $d=16$（有重叠）
- 使用带块内位置编码的 MLP 作为压缩函数 $\varphi$
- 压缩比约为 **16:1**（步长为 16）
- **目的**：以极低的计算成本扫描全局上下文，捕获粗粒度的语义信息

### 2. Token 选择（Selection）

基于压缩 token 的注意力分数选择最重要的 KV 块：

- 先计算查询与压缩 key 的注意力分数
- 选择注意力分数最高的 **top-16 个块**（每块 64 个 token）
- 包含 1 个固定的初始块和 2 个局部块
- **目的**：以 token 级精度关注关键信息，弥补压缩带来的信息损失

### 3. 滑动窗口（Sliding Window）

- 窗口大小 $w=512$
- 保留最近的 512 个 token
- **目的**：确保模型始终能精确感知局部上下文

### 硬件对齐的内核设计

NSA 的一大亮点是精心设计了硬件友好的 Triton 内核：

![NSA 内核设计示意图。内核按 GQA 组加载查询（Grid Loop），获取对应的稀疏 KV 块（Inner Loop），在 SRAM 上执行注意力计算。绿色块表示 SRAM 上的数据，蓝色表示 HBM 上的数据。](/assets/img/nsa/x3.png)
*图 2：NSA 内核设计。通过 GQA 组级加载和精细的循环调度，最大化 Tensor Core 利用率。*

关键优化：
- **块级内存访问**：利用连续内存布局，最大化 Tensor Core 利用率
- **GQA 组级共享**：消除冗余 KV 传输，不同查询头共享选择结果
- **平衡算术强度**：确保计算与内存访问的比值处于最优区间

## 关键实验结果

### 性能对比

实验在 **27B 参数**（3B 激活参数）的 GQA + MoE 模型上进行，预训练使用 270B token。

![Full Attention 与 NSA 的性能和效率对比。左图显示 NSA 在通用基准、长上下文任务和推理评估上的平均表现超越 Full Attention。右图展示在 64k 长序列上，NSA 在解码、前向和反向传播阶段均实现显著加速。](/assets/img/nsa/x1.png)

**通用基准结果**：NSA 在 9 项指标中的 7 项上超越 Full Attention，特别是推理任务表现突出（DROP: +0.042, GSM8K: +0.034）。稀疏注意力的预训练迫使模型聚焦最重要的信息，反而过滤了噪声。

## 深度思考

### 为什么重要？

1. **打破"稀疏=降级"的偏见**：NSA 是少有的在原生预训练中使用稀疏注意力且**超越**全注意力基线的工作。这表明稀疏性不仅不会损害性能，合理设计的稀疏模式甚至可以充当正则化，帮助模型聚焦关键信息。

2. **全链路加速**：不同于只优化推理的方法，NSA 同时加速了训练（前向/反向）和推理（预填充/解码），这对实际部署至关重要。

3. **硬件-算法协同设计**：NSA 的设计哲学是"先理解硬件，再设计算法"。通过分析算术强度在不同阶段的特性，针对性地优化——训练和预填充阶段减少计算量（compute-bound），解码阶段减少内存访问（memory-bound）。

### 局限性

1. **模型规模**：实验仅在 27B（3B 激活）模型上验证，在更大规模（如 671B 的 DeepSeek-V3）上的表现尚待验证。
2. **硬件特异性**：Triton 内核针对 A100 GPU 优化，在其他硬件（如 H100、TPU）上需要重新适配。
3. **超参数敏感性**：压缩块大小、选择块数量等超参数可能需要根据任务和模型规模调整。
4. **注意力压缩的信息损失**：虽然有选择机制补偿，但压缩不可避免地会损失部分细粒度信息。

### 对领域的影响

- **长上下文训练的加速器**：随着推理模型（如 o1、R1）需要越来越长的 chain-of-thought，NSA 的训练加速能力将变得至关重要。
- **稀疏注意力的范式转移**：从"推理时近似"转向"训练时原生稀疏"，可能成为未来注意力设计的标准范式。
- **DeepSeek 技术栈的拼图**：NSA 可能已被集成进 DeepSeek 的后续模型中，为其长上下文能力提供技术支撑。

## 总结

NSA 是一篇**工程与算法深度融合**的优秀工作。它通过三个层次的稀疏策略（压缩全局扫描 → 块级精确选择 → 滑窗局部覆盖）构建了一个完整的注意力信息漏斗，并通过硬件对齐的内核设计将理论加速转化为实际性能提升。最令人印象深刻的是，NSA 在保持甚至超越全注意力性能的同时，实现了高达 11.6× 的解码加速。这不仅仅是一个高效注意力变体，更是一种新的注意力设计范式——**先天稀疏，而非后天裁剪**。
