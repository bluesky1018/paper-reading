## 一句话总结

DeepSeek-V3.2 通过三大技术突破——**DeepSeek 稀疏注意力（DSA）**、**可扩展强化学习框架**和**大规模 Agent 任务合成流水线**——实现了与 GPT-5 可比的性能，其变体 DeepSeek-V3.2-Speciale 更是超越 GPT-5、比肩 Gemini-3.0-Pro，在 IMO 2025 和 IOI 2025 均获金牌。

## 背景与动机

开源与闭源模型之间的性能差距正在**扩大**而非缩小。通过分析，DeepSeek 团队识别出开源模型在复杂任务上的三个关键缺陷：

1. **架构效率瓶颈**：vanilla attention 在长序列上计算复杂度为 $O(L^2)$，严重制约可扩展部署和后训练
2. **后训练投入不足**：开源模型在 post-training 阶段的计算投入远低于闭源模型
3. **Agent 能力落后**：在泛化性和指令遵循方面明显不及闭源竞品

## 架构/方法详解

### 1. DeepSeek Sparse Attention (DSA)

DSA 是本文最核心的架构创新，将注意力复杂度从 $O(L^2)$ 降低到 $O(Lk)$，其中 $k \ll L$。

![图2：DeepSeek-V3.2 的 DSA 注意力架构](/assets/img/deepseek-v3.2/x2.png)
*图2：DSA 在 MLA 架构下的实例化示意图。绿色部分展示了 DSA 如何通过 indexer 选择 top-k 的 key-value 条目。*

DSA 由两个组件构成：

**闪电索引器（Lightning Indexer）**：计算 query token 与前文 token 之间的索引分数：

$$I_{t,s} = \sum_{j=1}^{H^I} w_{t,j}^I \cdot \text{ReLU}(\mathbf{q}_{t,j}^I \cdot \mathbf{k}_s^I)$$

- 使用 ReLU 激活函数（考虑吞吐量）
- 索引器头数少，可用 FP8 实现，计算效率极高

**细粒度 token 选择**：仅检索 top-k 索引分数对应的 key-value 条目，然后在稀疏选择的条目上计算注意力。

**训练流程**：
- **Dense Warm-up 阶段**：冻结所有参数，仅用 KL 散度损失训练 indexer（1000 步，2.1B tokens）
- **Sparse Training 阶段**：引入 token 选择机制，联合优化所有参数（15000 步，943.7B tokens），每个 query token 选择 2048 个 key-value tokens

![图3：推理成本对比](/assets/img/deepseek-v3.2/x3.png)
*图3：DeepSeek-V3.1-Terminus 与 V3.2 在 H800 集群上的推理成本对比——(a) 预填充和 (b) 解码阶段，V3.2 在长序列场景下成本显著降低。*

### 2. 可扩展强化学习框架：Scaling GRPO

DeepSeek-V3.2 的后训练计算预算**超过预训练成本的 10%**，这是一个极为大胆的投入。为了稳定如此大规模的 RL 训练，团队引入了四项关键技术：

- **无偏 KL 估计**：修正 K3 估计器，消除系统性估计误差，避免梯度不稳定
- **Off-Policy 序列掩码**：对 KL 散度超过阈值 $\delta$ 的负优势序列进行掩码，提高对 off-policy 更新的容忍度
- **Keep Routing**：保留推理阶段的 MoE 专家路由路径并在训练中强制执行，避免路由不一致导致的不稳定
- **Keep Sampling Mask**：保留 top-p/top-k 采样的截断掩码，确保训练和推理策略共享相同的动作子空间

**专家蒸馏 + 混合 RL 训练**：
- 先为每个领域（数学、编程、推理、Agent 等）训练专门的 specialist 模型
- 用 specialist 生成领域数据进行蒸馏
- 最后将推理、Agent、人类对齐训练合并到一个 RL 阶段，避免灾难性遗忘

### 3. Thinking in Tool-Use：Agent 能力突破

![图4：工具调用场景中的思考保留机制](/assets/img/deepseek-v3.2/x4.png)
*图4：思维上下文管理策略——仅在新用户消息到来时丢弃推理内容，工具输出期间保留推理上下文，避免冗余重推理。*

**思考上下文管理**：
- 当仅有工具消息时，保留推理内容
- 新用户消息到来时才清除推理内容
- 工具调用历史始终保留

**Cold-Start**：通过精心设计的 system prompt 将推理和工具使用统一到单一轨迹中。

**大规模 Agent 任务合成**：
- 自动合成了 **1,827 个任务导向环境**和 **85,000 个复杂 prompt**
- 环境包括：搜索 Agent、代码 Agent（从 GitHub 挖掘 issue-PR 对）、代码解释器、通用 Agent
- 合成流程：环境与工具集构建 → 任务合成 → 解决方案生成 → 验证函数验证

![图5：合成 Agent 数据的 RL 训练效果](/assets/img/deepseek-v3.2/synthesis-rl-plot.png)
*图5：仅在合成 Agent 数据上进行 RL 训练即可在 Tau2Bench、MCP-Mark、MCP-Universe 上大幅提升性能，而仅在代码和搜索场景训练无此效果。*

## 深度思考

### 为什么重要？

1. **缩小开闭源差距**：DeepSeek-V3.2 证明开源模型可以在推理、Agent 等核心能力上逼近甚至超越闭源前沿
2. **DSA 开创稀疏注意力新范式**：不同于 sliding window 等粗粒度方法，DSA 通过轻量级 indexer 实现细粒度 token 级选择，且在 MLA 框架下无缝集成
3. **RL 可扩展性的关键突破**：后训练计算超过预训练 10%+，四项稳定性技术（无偏 KL、Off-Policy 掩码、Keep Routing、Keep Sampling Mask）为大规模 RL 铺平道路
4. **合成 Agent 任务的泛化能力**：1,827 个合成环境训练出的能力可泛化到未见过的 benchmark，这对构建通用 Agent 意义重大

### 局限性

1. **世界知识深度不足**：由于总训练 FLOPs 较少，知识广度仍落后于 Gemini-3.0-Pro
2. **Token 效率低**：V3.2 需要更长的生成轨迹才能匹配 Gemini-3.0-Pro 的输出质量
3. **复杂任务仍有差距**：在 GPQA Diamond、HLE 等需要深度推理的任务上仍弱于 Gemini-3.0-Pro
4. **上下文长度限制**：128K 窗口在 Agent 场景中经常被用尽，需要额外的上下文管理策略

### 深远影响

- **Specialist → Distillation → Mixed RL** 的训练范式可能成为行业标准
- DSA 为未来百万级 token 上下文打开了可能性
- Agent 任务合成流水线可能催生"Agent Gym"生态
- RL 预算超过预训练 10% 的做法表明，后训练阶段的重要性正在快速上升

## 总结

DeepSeek-V3.2 是一份技术含量极高的工程杰作。三大技术支柱——DSA 稀疏注意力（将长序列效率提升一个量级）、可扩展 GRPO 框架（将 RL 规模推向预训练 10%+）、大规模 Agent 任务合成（1,827 个自动生成的环境）——共同将开源大模型推向了与 GPT-5 比肩的水平。

Speciale 变体在 IMO 2025、IOI 2025、ICPC WF 2025 上的金牌表现更是展示了"放开 token 限制后推理能力的上限"。这不仅是 DeepSeek 的里程碑，更是整个开源社区的里程碑——它证明了开源模型在 Agent、推理、代码等各个维度上都有能力与最强闭源模型竞争。
